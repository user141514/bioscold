# Style Profile

Merged from exemplar language patterns and JCIM scene norms.

| Style Dimension | Target Venue Expectation | Exemplar Pattern | Applied To This Paper |
|---|---|---|---|
| **Opening frame** | Computational framing (chemical space scale, algorithmic challenge), not biological motivation | Ertl (2020): bioisosterism as informatics challenge; Hussain & Rea (2010): "millions of compounds" scale problem | Current draft opens with "replacing a selected substituent...remains difficult to benchmark rigorously" — good computational framing. Consider stronger scale hook in first sentence. |
| **Introduction arc** | Broad importance → specific technical gap → contribution statement (2-3 paragraphs) | Gap stated as technical limitation ("existing approaches require explicit fragment definitions"), not generic need | Current draft maps field problem → three difficulties → two contributions. Pattern matched well. Gap narrowing could be sharper (three difficulties before contribution statement). |
| **Methods structure** | Two-part: representation first, then model; Methods as direct response to Introduction's stated gap | Ertl: auto feature learning addresses "needs expert definitions"; Hussain & Rea: algorithm directly solves computational complexity problem | Current draft: Section 3.1 task formulation → 3.2 benchmark → 3.3-3.5 methods → 3.6 evaluation → 3.7 A4C. Good logical flow. Methods clearly address the three Introduction difficulties. |
| **Results architecture** | Benchmark protocol → primary results → ablation → robustness → qualitative interpretation | All three exemplars: establish evaluation framework first, then present comparative table, then mechanistic analysis | Current draft: 4.1 overall result → 4.2 prior_ranks mechanism → 4.3 rescue/lost → 4.4 schema alignment audit → 4.5 A4C. Follows JCIM pattern. |
| **Statistical reporting** | Explicit operational metric definitions; CIs preferred over point estimates alone; bootstrap replication count stated | All exemplars define metrics operationally ("hit scored if any positive appears in top K") | Current draft: bootstrap B=5000 stated, paired differences with CIs, query-level resampling. Appropriate for JCIM. |
| **Comparative language** | "Improves over" / "competitive with" / "comparable to" — never "superior to" without formal test | Ertl: "comparable to expert-curated fragments"; Polishchuk: "competitive with existing generators" | Current draft: "improves over," "ΔTop-10 = +0.0686 with CI." Conservative. Good. Check for any residual "outperforms" / "state-of-the-art" language. |
| **Limitations disclosure** | Openly acknowledge scope limits; concrete future work tied to specific limitations; no vague "further research needed" | Polishchuk: explicitly states scope ("designed for R-group replacement, not scaffold hopping") | Current draft Section 5.6: six specific limitations, each concrete. Excellent. |
| **Closing** | Focused assessment of what method can/cannot do → concrete extensions → avoid broad applicability claims | Exemplars close with limitation-aware forward look | Current draft Conclusion lists four limitations that "define the scope." Good pattern match. |
| **Register** | Nominal constructions, explicit definitions, no promotional adjectives, abbreviation discipline | JCIM register is consistently precise, quantitative, anti-hype | Current draft: heavy use of nominal constructions ("we present," "we formalize"), quantitative throughout. Register is appropriate. |
| **Figure/table style** | Tables with clear headers, footnotes for deviations; figures show distributions not single points | All JCIM exemplars use structured tables with footnoted clarifications | Current draft tables (M1-M5, 1-4) are cleanly structured with footnotes. Good. |
