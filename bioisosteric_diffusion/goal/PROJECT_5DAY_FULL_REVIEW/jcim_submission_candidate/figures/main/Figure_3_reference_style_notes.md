# Figure 3 Reference Style Notes

## Scope

The requested literature scan was used only to extract layout discipline for a JCIM-style benchmark performance overview. No graphical assets, icons, colors, molecular drawings, or exact layouts were copied.

## Benchmark / Molecular ML References

| Reference | Link | Layout principle used for Figure 3 |
|---|---|---|
| GuacaMol: Benchmarking Models for De Novo Molecular Design | https://pubs.acs.org/doi/10.1021/acs.jcim.8b00839 | Use restrained benchmark performance comparison, compact labels, and clear metric context. |
| Most Ligand-Based Classification Benchmarks Reward Memorization Rather than Generalization | https://arxiv.org/abs/1706.06619 | Keep performance evidence separate from evidence-boundary interpretation. |
| Exposing the Limitations of Molecular Machine Learning with Activity Cliffs | https://pubs.acs.org/doi/10.1021/acs.jcim.2c01073 | Present aggregate performance in a controlled style and avoid overclaiming beyond the measured endpoint. |
| MoleculeNet: A Benchmark for Molecular Machine Learning | https://pubs.rsc.org/en/content/articlelanding/2018/sc/c7sc02664a | Use table-like benchmark organization and method/metric alignment for high information density. |

## Applied Decisions

- Use a horizontal forest/dot plot with CI bars instead of a bar chart.
- Add aligned evidence-role and delta columns to keep claim boundaries outside the data area.
- Use a hollow light-gray marker for Best-of-DE+HGB diagnostic so it does not look deployable.
- Keep annotations out of the plot area; evidence-role text is table-aligned.
- Use subtle group separators instead of decorative group blocks or legends.
